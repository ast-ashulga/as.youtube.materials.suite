# Cross-Reference Guide

Rules for building wiki-style navigation between session files.

---

## Link Syntax

All links use **relative paths** so the session folder is portable.

| From file | To file | Link syntax |
|-----------|---------|------------|
| `index.md` | report | `[Title](./reports/video-slug.md)` |
| `report` | another report | `[Related: Title](../reports/other-slug.md)` |
| `report` | `summary.md` | `[Session Summary](../summary.md)` |
| `report` | `sources.md` | `[All Sources](../sources.md)` |
| `summary.md` | report | `[Full Report: Title](./reports/video-slug.md)` |
| `sources.md` | report | `[Report](./reports/video-slug.md)` |

---

## Detecting Related Videos

Two videos are "related" and should cross-reference each other if they share:

1. **Same theme/topic cluster** — both cover the same sub-topic (e.g., "MCP servers" within a Claude session)
2. **Sequential content** — one video is a prerequisite or follow-up to another
3. **Contrasting perspectives** — two videos offer different viewpoints on the same question
4. **Same speaker or channel** — note as "Also from [Channel]"

Assign each video 1-3 theme tags during analysis (e.g., `["getting-started", "prompt-engineering"]`). Videos sharing any theme tag should cross-reference each other.

## index.md Structure

The index is the session landing page. Structure it as:

```markdown
# Research Session: [Topic(s)]

**Date**: YYYY-MM-DD · **Videos**: N included, M rejected · **Depth**: brief|standard|comprehensive

## Quick Navigation

- [Session Summary](./summary.md) — consolidated overview of all included videos
- [All Sources](./sources.md) — full list of candidates, included and rejected

---

## Videos by Theme

### [Theme 1 Name]

- [Video Title](./reports/video-slug.md) — [Channel] · [Duration] · [one-line description]
- [Video Title](./reports/video-slug.md) — [Channel] · [Duration] · [one-line description]

### [Theme 2 Name]

- ...

---

## All Included Videos (alphabetical)

| Title | Channel | Duration | Themes |
|-------|---------|---------|--------|
| [Title](./reports/slug.md) | Channel | MM:SS | tag1, tag2 |

---

*Generated by yt-research on YYYY-MM-DD*
```

## summary.md Structure

```markdown
# Session Summary: [Topic(s)]

[2-3 paragraph synthesis of the session's findings. What are the main themes? Key consensus points? Notable disagreements or tensions between sources?]

## Key Insights Across All Videos

- [Insight 1 — reference: [Video Title](./reports/slug.md)]
- [Insight 2 — reference: [Video Title](./reports/slug.md)]
- [5-10 bullets, each with source reference]

## Recommended Reading Order

For someone new to these topics, suggested sequence:

1. [Title](./reports/slug.md) — why start here
2. [Title](./reports/slug.md) — why second
3. ...

---

*[N videos included · M videos rejected · See [All Sources](./sources.md)]*
```

### summary.md — Bilingual Supplement (when lang includes Russian)

Append this section after the English content (including the footer line) when `session_config.lang` includes `ru`:

```markdown
---

## Краткое содержание сессии (на русском)

[2-3 абзаца синтеза результатов сессии на русском языке. Основные темы, ключевые выводы, заметные противоречия между источниками. Синтезируй из полей `russian_summary` всех включённых анализов — не копируй их дословно, а обобщи.]

## Ключевые выводы по всем видео

- [Вывод 1 — источник: [Название видео](./reports/slug.md)]
- [Вывод 2 — источник: [Название видео](./reports/slug.md)]
- [5–10 пунктов, каждый со ссылкой на источник]

## Рекомендуемый порядок просмотра

Для новичка в этих темах:

1. [Название](./reports/slug.md) — почему начать здесь
2. [Название](./reports/slug.md) — почему вторым
3. ...
```

## Bidirectional Reference Rule

If video A references video B as "related", video B **must** also reference video A. Build cross-references in the consolidation step when all videos are known. Do not add cross-references during individual analysis.
