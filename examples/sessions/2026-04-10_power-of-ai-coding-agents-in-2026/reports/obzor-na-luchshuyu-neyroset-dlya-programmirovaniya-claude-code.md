## Summary

This hands-on Russian-language review of Claude Code — Anthropic's terminal-based AI coding agent — covers a three-tier LLM hierarchy for developers: chat-based copy-paste (lowest leverage), in-editor assistants (mid-tier), and full agentic systems like Claude Code (highest leverage). Capabilities covered: image-to-website generation, bash command execution, project memory via CLAUDE.md, MCP integrations (GitHub, Docker, Figma, documentation), multi-agent setups with separate agents for Git, testing, and design, and the "Dangerously Skip Permissions" mode for fully autonomous operation. The author built a complete English-vocabulary web application in roughly 40 minutes without writing any code. The conclusion is measured: vibe-coding lowers the barrier to entry, but fundamental programming knowledge is still required for debugging, security, and quality control.

## Key Takeaways

- Claude Code is subscription-based ($20/month for Sonnet 4.5; $100/month adds Opus 4.1) — sees the entire codebase and acts as a fully autonomous developer
- The most powerful LLM-coding paradigm is the agentic tier — AI reads the entire codebase, commits to GitHub, pulls designs from Figma
- MCP allows Claude Code to connect to GitHub, Docker, Figma, and library documentation — dramatically expanding capabilities beyond code alone
- Multi-agent architectures are natively supported: separate agents for Git, testing, and design enable parallel specialized work
- The author built a complete English-learning web app with Google auth, ChatGPT translation, PostgreSQL, and a vocabulary trainer in ~40 minutes via vibe-coding
- Fundamental programming knowledge still required: AI-generated projects fail, get hacked, and need debugging without it
- Best practice: start with Opus 4.1 for architecture decisions, switch to Sonnet 4.5 for routine bug fixes to manage cost

## Action Items

- [ ] Install Node.js and Claude Code via terminal, then authenticate with your Anthropic subscription
- [ ] Use the `#` command to save persistent instructions into user-level or project-level memory
- [ ] Run `claude init` on an existing codebase to auto-generate a CLAUDE.md context file
- [ ] For complex projects, start with Opus 4.1 for architecture, then switch to Sonnet 4.5 for implementation to control cost
- [ ] Run Claude Code inside a Docker container when using Dangerously Skip Permissions mode for safety

## Key Terminology

| Term | Definition |
|------|-----------|
| Claude Code | Anthropic's terminal-based agentic AI coding tool, powered by Sonnet 4.5 and Opus 4.1 models |
| MCP | Model Context Protocol — allows Claude Code to control GitHub, Docker, Figma, and documentation services |
| Vibe-coding | Building software by giving natural-language prompts to an AI agent with minimal or no manual code writing |
| Dangerously Skip Permissions | Flag enabling fully autonomous operation — recommended only inside Docker containers for safety |
| CLAUDE.md | Project-specific context file generated by `claude init`, loaded into Claude's context on every session |
| Opus 4.1 vs Sonnet 4.5 | Opus 4.1 for complex architecture tasks ($100/month); Sonnet 4.5 for routine tasks ($20/month) |
